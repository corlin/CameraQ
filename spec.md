# 实时摄影构图智能助手

一句话定位：

**在用户拍摄照片/视频时，实时识别画面主体、结构、光线和构图问题，用极简提示帮助用户移动机位、调整取景、避开低级构图错误，并逐步学习用户审美。**

这个产品是可行的，但路线要稳：**先做“实时构图纠错”，再做“构图推荐”，最后做“审美个性化”。**

------

## 1. 产品不要一上来做“大师审美”，先做“低级错误消除器”

摄影构图里最容易产品化的，不是“高级美感”，而是新手常犯的错误：

地平线歪。
主体太小。
主体太死居中。
人物头顶空间太大。
手脚被切。
背景太乱。
人物脸太暗。
人物视线方向没有留白。
建筑线条歪斜。
画面没有前景、中景、背景层次。
视频拍摄时主体跑出安全框。

这些问题是**可检测、可解释、可反馈、用户能立刻感知价值**的。

所以第一阶段不要说：

> AI 帮你拍出高级审美大片。

而是说：

> 拍摄时实时提醒你：往左一点、后退一点、放低机位、修正水平线、避开背景干扰。

这个更容易落地，也更容易形成用户信任。

------

## 2. 整体技术架构

建议拆成 7 层。

```text
摄像头实时画面
   ↓
视觉感知层：YOLO / 分割 / 姿态 / 人脸 / 显著性
   ↓
画面结构层：地平线、直线、消失点、对称、明暗重心、色彩重心
   ↓
构图规则层：三分法、居中、留白、头顶空间、引导线、框中框、裁切安全区
   ↓
场景判断层：人像 / 风景 / 街拍 / 美食 / 产品 / 建筑 / 宠物 / Vlog
   ↓
建议决策层：只输出当前最重要的一条建议
   ↓
取景器交互层：辅助线、箭头、推荐框、语音/震动/文字提示
   ↓
用户反馈层：保存、删除、分享、手动选择，用于个性化学习
```

YOLO 适合放在第一层。Ultralytics YOLO11 官方支持目标检测、实例分割、分类、姿态估计、旋转框检测等任务，适合作为实时视觉理解底座。([Ultralytics Docs](https://docs.ultralytics.com/models/yolo11?utm_source=chatgpt.com)) 但构图判断还需要显著性、地平线、直线、空间布局、审美评分等模块配合。Apple Vision 官方也提供图像/视频分析能力，并有显著性区域分析、人像姿态检测等相关能力；Google MediaPipe 也提供移动端人体姿态、面部关键点等任务，适合移动端实时视觉场景。([Apple Developer](https://developer.apple.com/documentation/Vision/highlighting-areas-of-interest-in-an-image-using-saliency?utm_source=chatgpt.com))

## 3. 第一版 MVP：只做 6 个最确定的功能

第一版目标：**让用户打开相机后，30 秒内感受到“它真能指导我拍得更好”。**

不要做太多规则，先做最稳定的 6 个。

### 功能 1：主体识别与主体框

识别人、宠物、车辆、食物、产品、建筑等常见主体。

输出：

“当前主体是人物。”
“当前主体是食物。”
“当前主体是建筑。”

技术上用 YOLO 检测，必要时加 segmentation，获得主体轮廓。

价值：后续所有构图建议都依赖“谁是主体”。

------

### 功能 2：三分法 / 居中构图提示

判断主体是否靠近三分线、三分交点或画面中心。

但要注意：不是永远推荐三分法。

规则应该是：

单人头像、人像半身：优先三分法或轻微居中。
证件感、产品图、建筑对称：优先居中。
风景大场景：优先地平线三分。
极简画面：允许大面积留白。

输出示例：

“人物可以向左一点，落在右三分线会更自然。”

“这个建筑适合居中构图，保持左右对称。”

“天空很漂亮，地平线可以放低一点。”

三分法、留白、画面平衡等是常见摄影构图基础，Adobe 的摄影构图指南也把三分法、平衡、留白、景深、裁切等作为核心构图实践。([Adobe](https://www.adobe.com/creativecloud/photography/technique/composition.html?utm_source=chatgpt.com))

------

### 功能 3：地平线 / 建筑线校正

这是最容易让用户感知价值的功能。

检测：

地平线是否倾斜。
建筑竖线是否歪。
道路、海平面、桌面是否歪。

输出示例：

“画面向右倾斜 2.5°，稍微顺时针调整。”

“建筑竖线外扩，建议后退一点或抬正手机。”

这一项不一定靠 YOLO，可以用传统视觉算法：

Hough Line
LSD 直线检测
边缘检测
透视线分析
陀螺仪辅助

------

### 功能 4：人物裁切安全区

这是人像拍摄刚需。

检测：

头顶空间是否过大。
头顶是否被切。
手、脚、肘部是否尴尬裁切。
脸是否太靠边。
人是否顶到画面边缘。

输出示例：

“后退半步，脚快被切掉了。”

“头顶空间太多，镜头稍微向下。”

“人物脸太靠左，右边留白不足。”

技术上可以用 YOLO-Pose、MediaPipe Pose 或 Apple Vision 人体姿态能力。MediaPipe Pose Landmarker 可输出人体关键点的图像坐标和 3D 世界坐标；Apple Vision 也支持人体姿态关键点检测。([Google for Developers](https://developers.google.com/edge/mediapipe/solutions/vision/pose_landmarker?utm_source=chatgpt.com))

------

### 功能 5：背景干扰检测

这个功能非常有产品感。

检测：

人物头顶是否有杆子、树枝、灯牌穿过。
背景高亮区域是否抢走主体注意力。
主体附近是否有杂乱物体。
人脸附近是否有强干扰线条。

输出示例：

“人物头顶有背景干扰，向左移动一步。”

“背景太亮，人物不突出，换个角度会更好。”

这里可以结合：

YOLO 检测干扰物。
显著性模型判断视觉注意力是否被背景吸走。
主体与背景的重叠关系。
亮度/对比度分布。

Apple Vision 的显著性区域示例强调可量化和可视化人们可能关注的图像区域，这类能力正好适合判断“观众注意力是否被主体之外的区域抢走”。([Apple Developer](https://developer.apple.com/documentation/Vision/highlighting-areas-of-interest-in-an-image-using-saliency?utm_source=chatgpt.com))

------

### 功能 6：拍后最佳裁切推荐

这是第一版最容易商业化的功能。

拍摄完成后，自动给用户 3 个推荐裁切：

经典三分法版。
居中高级版。
社交媒体竖屏版。

输出示例：

“推荐裁切 1：人物更突出。”

“推荐裁切 2：保留更多环境氛围。”

“推荐裁切 3：适合小红书/抖音封面。”

这个功能比实时移动建议更稳，因为拍后有更多时间计算，可以调用更重的审美模型。

Google 的 NIMA 研究就是让模型预测人类对图片质量和审美的评分分布，并可用于图像排序和摄影管线中的质量评估。([Google Research](https://research.google/blog/introducing-nima-neural-image-assessment/?utm_source=chatgpt.com))

## 4. MVP 技术方案

第一版可以这样做。

### 端侧实时模块

建议优先做手机端。

iOS：

Camera + AVFoundation
Core ML
Vision Framework
Metal 加速
SwiftUI / UIKit 取景器叠加层

Android：

CameraX
TFLite / MediaPipe / NCNN / ONNX Runtime Mobile
GPU Delegate / NNAPI
OpenGL / Compose UI

跨平台可以用 Flutter / React Native 做业务层，但实时相机和推理最好走原生模块。

------

### 模型组合

第一版不需要大模型。

建议组合：

YOLO 检测模型：识别人、车、宠物、食物、杯子、产品、建筑相关物体。
YOLO-Seg 或轻量分割模型：拿主体轮廓。
Pose 模型：做人像关键点、手脚裁切判断。
Face 模型：做人脸位置、视线方向、脸部明暗判断。
Saliency 模型：判断视觉注意力重心。
传统视觉算法：直线、地平线、消失点、对称轴。
轻量审美评分模型：拍后评分，不必实时每帧跑。

第一版实时推理不建议所有模型每帧都跑。

更合理的是：

YOLO：5-10 fps
Pose / Face：5-10 fps，仅人像场景启动
地平线检测：10-15 fps
UI 叠加：30/60 fps
审美评分：拍后或低频触发

这样手机不会太热，延迟也能控制。

## 5. 构图规则如何工程化

不要把摄影理论停留在文字层面，要变成指标。

### 指标 1：主体位置分

计算主体中心点与推荐构图点的距离。

推荐点包括：

中心点
左三分交点
右三分交点
上三分线
下三分线

分数示例：

```text
主体位置分 = 100 - 主体中心到推荐点的归一化距离 × 权重
```

------

### 指标 2：主体占比

不同场景有不同主体占比。

人像半身：主体高度可占画面 55%-80%。
全身人像：人物整体高度可占 65%-90%。
产品图：主体可占 50%-75%。
风景：主体可小一些，强调环境。
美食：主体可大一些，强调细节。

输出：

“主体太小，靠近一点。”

“主体太满，后退一点。”

------

### 指标 3：头顶空间

人像里非常关键。

计算：

头顶到画面上边缘距离
脸部中心到画面上边缘距离
身体关键点是否完整
眼睛是否在上三分线附近

输出：

“头顶空间太多，镜头向下。”

“头快顶到边了，稍微后退。”

------

### 指标 4：视线 / 运动留白

人物脸朝右，右侧应该留更多空间。
车往左开，左侧应该留更多空间。
宠物朝某方向看，也应该给视线方向留白。

输出：

“人物看向右侧，右边留白再多一点。”

这会比机械三分法更高级。

------

### 指标 5：地平线稳定

计算主水平线角度。

输出：

“向左旋转一点。”

“地平线已经水平。”

这可以做成最直观的取景器辅助线。

------

### 指标 6：背景干扰分

计算主体周围是否存在高显著、高亮、高对比、强线条干扰。

尤其是：

头顶竖线。
脸部旁边高亮物。
主体后方复杂纹理。
主体边缘和背景粘连。

输出：

“背景有干扰，向右移动一步。”

------

### 指标 7：画面平衡分

计算视觉重量：

主体面积
主体亮度
主体颜色饱和度
背景高亮区域
显著性区域分布

判断画面是否左重右轻、上重下轻。

输出：

“右侧画面太空，可以把主体稍微右移。”

------

## 6. 场景模式设计

第一版至少要有 6 个场景模式。

### 人像模式

重点规则：

脸部位置
眼睛高度
头顶空间
手脚裁切
背景干扰
视线留白
人物与背景分离

核心提示：

“后退一点，保留完整身体。”

“人物眼睛放到上三分线更自然。”

“背景干扰人物头部，换个角度。”

------

### 风景模式

重点规则：

地平线
天空/地面比例
前景层次
引导线
消失点
画面纵深

核心提示：

“地平线放在下三分线，天空更有气势。”

“前景太空，可以找一个石头/树枝作为前景。”

“道路形成引导线，机位低一点更有纵深。”

------

### 建筑模式

重点规则：

垂直线
对称轴
消失点
中心构图
边缘留白
畸变控制

核心提示：

“建筑适合居中，保持左右对称。”

“竖线倾斜明显，手机抬正一点。”

“后退一点，保留建筑顶部。”

------

### 美食模式

重点规则：

主体占比
桌面水平
色彩对比
俯拍/斜拍建议
杂物干扰
边缘裁切

核心提示：

“盘子被切边了，稍微后退。”

“桌面有点歪，手机顺时针调整。”

“主体可以再靠近一点，食物更有冲击力。”

------

### 产品模式

重点规则：

主体居中
边缘完整
背景简洁
高光反射
文字可见
透视稳定

核心提示：

“产品偏左，居中会更像商业图。”

“背景杂物较多，建议换纯色背景。”

“产品边缘快被切掉，后退一点。”

------

### Vlog / 视频模式

重点规则：

人物安全框
脸部追踪
运动方向留白
镜头稳定
主体持续居中或三分
画面抖动
最佳片段选择

核心提示：

“人物快出画了，镜头向右跟一点。”

“脸部太靠下，手机抬高一点。”

“这 3 秒构图最好，可以作为封面。”

------

## 7. 建议决策：一次只提醒一件事

这个产品成败很大程度取决于交互。

拍摄时用户没有耐心看复杂分析。

所以每一帧虽然可以算很多指标，但最终只能输出一个最高优先级建议。

优先级建议：

第一优先级：硬错误
比如人脸被切、主体跑出画面、地平线严重歪。

第二优先级：主体问题
比如主体太小、太靠边、没有留白。

第三优先级：背景干扰
比如头顶长杆子、背景抢戏。

第四优先级：构图优化
比如三分法、居中、引导线、对称。

第五优先级：审美增强
比如建议降低机位、换角度、加入前景。

这样用户不会被提示轰炸。

## 8. 取景器 UI 设计

建议取景器里只保留 4 类元素：

### 1. 轻量辅助线

三分线
中心线
水平线
安全框

默认半透明，不要太吵。

------

### 2. 推荐落点

比如在画面上显示一个柔和的圆点：

“把人物脸移动到这里。”

或者显示一个推荐框：

“让主体进入这个框。”

------

### 3. 方向提示

简单箭头：

向左
向右
靠近
后退
抬高
降低
旋转校正

提示文案必须短：

“向右一点。”

“后退半步。”

“手机放低。”

“地平线拉平。”

------

### 4. 拍后评分和裁切

拍完后不要只给分，要给原因：

“构图 82 分：主体清晰，背景略杂。”

“推荐裁切：人物更突出。”

“封面建议：第 4.2 秒。”

## 9. 数据闭环怎么做

第一版不要指望公开数据集解决全部问题。

公开数据只能做预训练和参考。

CADB 是图像构图质量评估数据集，目标是评估图像整体构图质量；PICD 则是较新的摄影构图分析数据集，包含多种构图类别和场景，用于构图理解、审美质量评估、构图感知裁切等任务。([GitHub](https://github.com/bcmi/Image-Composition-Assessment-Dataset-CADB?utm_source=chatgpt.com)) CVPR 2025 的 PICD 论文也指出，构图理解仍有明显挑战，专用模型和多模态大模型都还容易受到语义干扰，不能简单认为“丢给多模态大模型就能懂构图”。([CVF Open Access](https://openaccess.thecvf.com/content/CVPR2025/html/Zhao_Can_Machines_Understand_Composition_Dataset_and_Benchmark_for_Photographic_Image_CVPR_2025_paper.html?utm_source=chatgpt.com))

真正有价值的数据来自用户拍摄过程。

建议埋点：

用户拍了几张。
哪张保存了。
哪张删除了。
哪张分享了。
用户是否采纳了提示。
提示后构图分是否提高。
用户最终选择了哪个裁切版本。
用户偏好人像大头、环境人像、极简留白还是强对称。

最重要的数据不是“单张好照片”，而是：

**同一场景下，坏构图 → 调整过程 → 好构图 的轨迹数据。**

这个才是未来训练“实时构图推荐模型”的金矿。

## 10. 迭代路线图

### 阶段 0：构图规则库建设，2-4 周

目标：把摄影知识变成算法指标。

输出物：

构图规则清单
场景分类规则
评分指标定义
提示文案库
MVP 原型流程图

第一批规则建议控制在 20 条以内，不要贪多。

优先级：

人像头顶空间
手脚裁切
主体占比
主体位置
地平线
三分法
居中构图
背景干扰
视线留白
画面倾斜
产品边缘完整
建筑对称

------

### 阶段 1：离线图片分析 Demo，4-6 周

先不做实时相机。

先让用户上传一张照片，系统输出：

主体识别
构图评分
问题诊断
推荐裁切
改进建议

这是最便宜的验证方式。

例如：

上传一张人像。

系统输出：

“人物头顶空间过大。”
“背景高亮区域抢戏。”
“建议裁切为 4:5。”
“人物脸部可落在左上三分点。”

这个阶段可以用 Python + OpenCV + YOLO + 简单前端快速做出来。

验证问题：

用户觉得建议准不准？
用户愿不愿意按建议裁切？
评分是否有解释力？
不同场景是否能识别？

------

### 阶段 2：实时取景器 MVP，6-10 周

开始做手机端实时相机。

第一版只做：

主体框
三分线
水平线
人像安全框
一句话提示
拍后推荐裁切

不要做复杂审美评分。

目标体验：

用户打开相机，对准人物。

系统提示：

“后退一点，保留完整身体。”

用户后退后，提示消失，变成：

“构图不错。”

这就已经是一个可展示 Demo。

## 13. 核心评分体系

建议不要只给一个“审美分”。

应该拆成 5 个分：

```text
主体分：主体是否清晰、大小是否合适、是否完整
结构分：水平线、对称、引导线、空间层次
平衡分：左右/上下视觉重量是否稳定
干扰分：背景是否抢戏，主体边缘是否干净
风格分：是否符合当前模式，如人像、风景、美食、产品
```

最终综合：

```text
构图总分 = 主体分 × 0.30
        + 结构分 × 0.25
        + 平衡分 × 0.20
        + 干扰分 × 0.15
        + 风格分 × 0.10
```

但用户界面里不要展示公式。

用户只看到：

“构图 82 分。”

“主体清晰，背景略杂，建议向左移动一步。”

## 16. 最大技术难点

### 难点 1：主体到底是谁

画面里有很多人、很多物体时，谁是主体？

解决办法：

最大人脸优先。
离中心最近的目标优先。
用户点击选择主体。
追踪用户选择的主体。
结合显著性判断。

第一版必须允许用户点选主体。

------

### 难点 2：审美规则有时会互相冲突

三分法建议往右。
背景干扰建议往左。
人物留白建议往右。
水平线建议旋转。

所以要有建议优先级。

永远优先解决硬错误：

主体完整 > 水平稳定 > 背景干扰 > 构图优化 > 风格增强。

------

### 难点 3：提示不能乱跳

实时画面轻微移动，系统提示不能每秒变 5 次。

解决办法：

加入时间平滑。
同一建议至少保持 1-2 秒。
分数变化超过阈值才切换建议。
拍摄中减少长文本，只保留方向提示。

------

### 难点 4：手机发热和耗电

不要每帧跑大模型。

解决办法：

轻模型端侧实时。
重模型拍后分析。
动态降帧。
场景触发式推理。
用户按下半快门/准备拍摄时提高推理频率。

这些先不要做：

不要一开始就训练大规模审美模型。

先把“拍摄时一句话指导”做好。

## 19. 产品成败关键

这个产品的关键不是算法多强，而是：

**提示是否准确。**

**提示是否及时。**

**提示是否简单。**

**用户照做之后，照片是否肉眼可见变好。**

所以建议你把第一版核心指标定为：

提示采纳率。
采纳后构图分提升。
用户是否保存推荐裁切。
用户是否愿意打开它作为默认相机。
用户是否愿意为人像/产品/旅行/视频模式付费。

不要只看模型 mAP、FPS、审美分相关性。

那些是技术指标，不是产品指标。

------

## 20. 最终建议

这款产品应该按下面这个核心逻辑推进：

```text
第一步：用 YOLO 看懂主体
第二步：用规则引擎发现低级构图错误
第三步：用实时 UI 给出一个最重要的动作建议
第四步：拍后给出最佳裁切和解释
第五步：用用户选择反向学习审美偏好
第六步：扩展到视频、AR眼镜、短视频创作、电商拍摄
```

最现实的第一版宣传语可以是：

> 一个会在取景器里提醒你的 AI 摄影老师。
> 不教玄学，只告诉你：往哪移、退几步、怎么裁，照片立刻更顺眼。

这个方向值得做。

但一定要记住：

**YOLO 解决“看见什么”，构图规则解决“哪里不对”，审美模型解决“整体好不好”，用户反馈解决“你喜欢什么”。**

这四层合起来，才是真正的实时摄影构图智能助手。
